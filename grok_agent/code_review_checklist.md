# AI Code Review Checklist

Use this checklist when reviewing code generated by Grok, kimi-cli, or any AI.

## General
- [ ] Code compiles cleanly (desktop + Emscripten)
- [ ] No new warnings
- [ ] Follows existing style and naming conventions
- [ ] Comments explain *why*, not just *what*

## Architecture
- [ ] Respects Milkdrop-style patterns (e.g. dual framebuffer ownership)
- [ ] No unnecessary new classes if simpler solution exists
- [ ] Proper resource cleanup (FBOs, shaders, textures)

## Performance
- [ ] No unnecessary GPU state changes
- [ ] FBO usage is efficient
- [ ] Works at 60fps on target hardware

## Robustness
- [ ] Handles edge cases (rapid switching, invalid textures, aspect ratio 1:1 and extreme)
- [ ] Fallbacks exist where needed
- [ ] Emscripten/WASM tested

## Testing
- [ ] All affected transition shaders visually verified
- [ ] No regressions in existing functionality
